1. Краткое введение в ASP на примере задачи.

Структура программы на языке ASP состоит из фактов, правил и исключений. Дадим описания и определения данным понятиям:

a. Факт: A.

b. Правило: A :- L1, …, Ln.

c. Ограничение целостности: :- L1, …, Ln.

Факт – всегда истинное утверждение.

Правило – утверждение, зависящее от истинности других утверждений.

Ограничение целостности – некоторый «фильтр», который задаёт ложные ситуации. По сути это сокращение от правила вида: false :- L1, …, Ln.

Рассмотрим на примере, каким образом строится программа на ASP. Пусть у нас есть несколько людей, между которыми установлены родственные связи:

person(jhon; max; jack; emma; tom; harry).

parent(jhon,(tom;max)).

parent(jack,(emma; harry)).

Таким образом, у нас есть двое родителей – jhon и jack – у которых есть по двое детей. Теперь для удобства установим пол каждого из людей:

male(jhon;max;jack;tom;harry).

female(emma).

Как видно, все вышеописанные факты определяют отношения между объектами, в нашем случае это родственные связи, и их свойства – пол каждого человека.

Целью данной программы будет являться поиск человека, у которого есть сестра. Для этого нужно создать правило, позволяющее решить задачу:

havesister(X) :- person(X),parent(Y, X),parent(Y, Z), female(Z), X != Z.

Данное правило формально звучит следующим образом: сестра есть у того, кто является ребёнком родителя, у которого есть ещё один ребёнок женского пола, и имена его детей различны.

Чтобы запустить эту программу в clingo, добавим строку для вывода реузльтата:

#show havesister/1.

Результат в clingo:

Answer: 1

havesister(harry)

SATISFIABLE
